# Loom Frontend Baseline

## 基线范围

当前前端基线覆盖：

- 主题系统
- 标题栏与全局壳层
- 左侧导航 / 项目 / 会话树
- 会话区与输入框
- 右侧轨迹栏
- 设置页模型管理

能力、文件、记忆、自动化等页面本轮继承统一 shell，但不在本次基线内做同等深度的专页设计。

## 当前设计基线

### 主题

- 支持 `system / dark / light`
- 根节点使用 `data-theme`
- token 定义位于 `apps/web/src/styles/loomDesignTokens.css`

### 核心样式入口

- `apps/web/src/index.css` 是核心壳层样式入口
- 所有页面优先通过共享 class 与 token 接入
- 禁止继续在单页内散落局部视觉体系

### 组件关系

- 标题栏：全局控制中心
- 左栏：切换结构
- 主面板：结果与操作主场
- 右栏：补充信息与深入查看
- 弹层：只承载短时任务，如新增/编辑模型

## 交互基线

### 会话

- assistant 回复采用单位置连续过渡
- loading、思考、正文不再拆成多张卡片闪现
- 思考默认弱提示，展开到右栏

### 输入框

- 与主面板同表面体系
- 只有一个主发送动作
- 错误信息就近展示

### 设置页

- 模型列表常驻
- 新增 / 编辑通过弹层
- 高级配置默认折叠
- 切换、测试、编辑全部在卡片级完成

## 维护规则

- 任何新增 UI 都要先检查 `docs/ui-principles.md`
- 任何影响核心壳层的修改都要同步更新本文件和 `docs/loom-phase1-ui-spec.md`
- 如果为了快速交付引入临时样式，必须在同一轮清理或记录债务，不能长期保留
