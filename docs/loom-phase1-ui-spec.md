# Loom Phase 1 UI 规范

## 目标

Phase 1 的目标不是把每个页面都做复杂，而是先把核心壳层统一好，让标题栏、左栏、会话区、输入框、右栏、设置页和主题系统形成稳定基线。

## 视觉方向与品牌表达

- 视觉关键词：产品感、克制、清晰、可持续。
- 品牌表达：通过品牌标识、焦点色、主操作和轻微动效建立识别度。
- 禁止项：高饱和大面积背景、过多彩色按钮、说明文字霸屏、孤立的局部设计语言。

## 主题系统定义

- 主题模式：`system / dark / light`
- 默认行为：跟随系统
- 用户覆盖：写入本地存储并立即生效
- 切换入口：标题栏右侧操作区
- 技术约束：根节点挂 `data-theme`，明暗主题均使用统一语义 token

## token 结构

- 基础 token：字体、圆角、间距、阴影、动效时长
- 语义 token：`bg-*`、`line-*`、`text-*`、`brand-*`、`status-*`
- 组件引用 token，不直接写死页面颜色
- 新 token 必须同时考虑亮暗主题

## 组件层级

### 壳层

- 标题栏：品牌、数据源状态、主题切换、自动化状态
- 左栏：主导航、项目/会话树、系统入口
- 主面板：会话或工具页内容
- 右栏：轨迹、步骤、思考详情

### 会话系统

- 会话头部：标题、模式选择、项目归属
- 消息流：用户消息、同位过渡中的 assistant 消息、思考弱提示
- 输入区：工作台式输入卡片，状态统一

### 设置系统

- 常驻模型列表
- 新增/编辑弹层
- 高级配置折叠区
- 卡片级操作：测试、编辑、切换

## 页面适配规则

- 会话页使用最大阅读宽度，避免消息铺满整屏
- 工具页与设置页继承同一套 panel 体系
- 亮色主题优先保证层级和对比，不追求“纯白后台”
- 移动端在窄宽度下优先保留主面板内容，右栏可折叠，左栏收起

## 新 UI 接入 checklist

1. 是否复用了现有 token，而不是新增私有颜色？
2. 是否与标题栏、左栏、主面板属于同一产品语言？
3. 是否只有一个真正的主操作？
4. 是否把解释文字放到了用户即将决策的位置，而不是页面顶部？
5. 是否同时验证了暗色、亮色和系统跟随？
6. 是否考虑了 hover、focus、disabled、loading、error 状态？
