## 1. 测试目标

本计划用于覆盖 loom Phase 1 的三类核心风险：

- 会话主链路从壳层展示切换到真实数据时是否回归
- Trace / Context / Settings 等右侧与工具面板是否与合同一致
- 并发开发与开发期生产调试过程中，是否仍可验证、可回退、可留痕

## 2. 测试阶段

### 2.1 基线阶段

目标：

- 确认当前 shell/docs/contracts 现状可以作为并发开发起点

检查项：

- `apps/server` 测试可通过
- `apps/web` 构建可通过
- `packages/contracts` 类型可消费
- 本地 fallback 模式仍可打开工作台

### 2.2 合同冻结阶段

目标：

- 对 `ARC-001` ~ `ARC-005` 的产物做字段级审查

检查项：

- 领域对象定义完整
- REST 接口路径、方法、字段、错误口径明确
- SSE 事件名、事件载荷、完成态/失败态口径明确
- 分页、时间字段、排序口径明确

输出物：

- 联调检查单
- 接口样例
- 事件样例

### 2.3 并发开发阶段

目标：

- 保证前后端在冻结合同上并发推进时不漂移

检查项：

- 后端新增接口具备集成测试
- 前端关键路径具备冒烟测试或手测清单
- Context / Trace / Settings 数据结构与合同一致
- 任何字段变动均先更新文档再更新实现

### 2.4 联调与发布阶段

目标：

- 验证本地、集成分支和生产机 smoke 路径都可通过

检查项：

- 会话主链路连通
- SSE 事件顺序稳定
- 断线重连和空态/错态可接受
- 生产验证前已有回退方案

## 3. 测试范围

### 3.1 后端

- Project / Conversation / Message 读写接口
- Settings / Capability 读取接口
- Context 快照读取接口
- Trace / Run / RunStep 查询接口
- SSE 流接口及关键事件

### 3.2 前端

- 欢迎页进入工作台
- 会话列表切换与模式切换
- 消息列表加载与 composer 输入
- Trace / Context 右侧面板切换
- Capabilities / Settings 页面读取真实数据
- fallback 与远端读取切换

### 3.3 生产机验证

- Web 首页可访问
- `/api/health` 返回正常
- `/api/nodes` 返回正常
- 本轮新增关键 Phase 1 接口具备最小 smoke

## 5. 验证矩阵

| 阶段 | 目标 | 必须产物 | 通过标准 | 责任人 |
| --- | --- | --- | --- | --- |
| 基线冻结 | 确认当前工作树和基础构建可作为并发起点 | 基线检查记录、构建结果、测试结果 | 后端测试、前端构建、contracts 检查全部通过 | PM / QA |
| 合同冻结 | 锁定需求对象、接口、事件、错误口径 | 合同冻结清单、接口样例、事件样例 | REST / SSE / 数据模型都已明确，无歧义字段 | 架构师 / QA |
| 并发开发 | 让前后端在同一合同上并行推进 | 联调检查单、功能分支、接口测试结果 | 每个 lane 都能输出独立证据，且无合同漂移 | 后端 / 前端 / QA |
| 联调验证 | 验证真实接口与前端接线 | 联调记录、问题列表、修复记录 | 主链路跑通，错误态和空态可解释 | QA / PM |
| 生产验证 | 验证 release 与 smoke 路径 | smoke 记录、回退记录、生产机操作记录 | Web 首页、健康检查、节点接口通过，且可回退 | PM / QA |

## 6. 角色责任

| 角色 | 责任 |
| --- | --- |
| QA | 维护计划、执行联调检查、出具测试结论 |
| 后端开发 | 提供接口测试、错误态回归证据 |
| 前端开发 | 提供冒烟测试、页面状态证据 |
| PM | 审核测试是否满足进入联调和生产验证门槛 |

## 7. 结果沉淀要求

每轮测试至少沉淀以下内容：

- 测试范围
- 执行环境
- 执行结果
- 已知问题
- 是否允许进入下一阶段

测试结论可以维护在测试分支文档中，但在合并前必须回填到需求池或 PR 描述。

## 8. 需要前后端补齐的测试证据

- 前端：关键页面截图或录屏、构建成功记录、主流程冒烟结论、SSE 接线结果。
- 后端：接口测试结果、集成测试结果、SSE / Trace 事件样例、错误态样例。
- 联调：字段对齐表、问题列表、修复回归记录、最终通过结论。
